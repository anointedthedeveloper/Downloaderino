- Additionally, TypeScript is enforcing verbatim module syntax rules, so imports of types must be declared with the type-only form (import type ...). Those TS1484 errors point to that.

Files to check in your repo:
- package.json: https://github.com/anointedthedeveloper/Downloaderino/blob/main/package.json
- example source using types-only imports: src/App.tsx — https://github.com/anointedthedeveloper/Downloaderino/blob/main/src/App.tsx

**Solution**

1) Verify runtime dependencies
- First confirm that react and react-dom are listed in package.json dependencies (open https://github.com/anointedthedeveloper/Downloaderino/blob/main/package.json and check the "dependencies" section). If they are missing, add them (npm install react react-dom) before proceeding.

2) Add TypeScript React type declarations
- Install the type declaration packages as devDependencies so TypeScript can find JSX and React types:

```bash
# from repo root
npm install --save-dev @types/react @types/react-dom
```

(or use your package manager: yarn add -D @types/react @types/react-dom)

3) Fix type-only import errors (TS1484)
- Update imports that only bring in TypeScript types to use the type-only syntax. For example, in src/App.tsx change imports like:

```ts
// before
import { MovieItem, MovieDetail, LinksResponse } from './types';
```

to

```ts
// after
import type { MovieItem, MovieDetail, LinksResponse } from './types';
```

Do this for other files flagged by TS1484 (the log lists many, e.g. src/components/MovieCard.tsx, src/components/MovieDetailView.tsx). Use `import type ...` whenever you import only types.

4) Ensure tsconfig JSX setting (if applicable)
- Open tsconfig.json and ensure the JSX transform is set appropriately for your React version. For React 17+ with the new JSX transform, use:

```json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "skipLibCheck": true
  }
}
```

- Adding "skipLibCheck": true can reduce noisy type errors from node_modules while fixing the core issue. If your project target is older or you intentionally use the classic runtime, set "jsx" to "react" instead.

5) Re-run build locally and on Netlify
- After making the above changes, run locally:
  - npm ci (or npm install)
  - npm run build
- Then push commits to trigger your Netlify build.
